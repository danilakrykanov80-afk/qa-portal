# Tests

## Test 1 - Главная и каталог сервисов загружаются [sanity]
Viewport: desktop
Description: Проверяет, что публичная главная страница открывается и отображает карточку сервиса.

Steps:
1. Открыть главную страницу сайта.
2. Дождаться полной загрузки интерфейса.
3. Убедиться, что виден заголовок "Сервисы".
4. Убедиться, что карточка BotHunter отображается.

Expected: Главная страница загружается без ошибок и показывает сервис BotHunter.

## Test 2 - Выбор соцсети для BotHunter доступен
Viewport: desktop
Description: Проверяет переход с карточки сервиса на экран выбора соцсети.

Steps:
1. На главной странице нажать "Открыть сервис" у BotHunter.
2. Дождаться открытия страницы сервиса.
3. Проверить наличие карточек ВК, ТГ, ОК и MAX.

Expected: Пользователь видит экран выбора соцсети с четырьмя доступными вариантами.

## Test 3 - Переход в разделы выбранной соцсети работает
Viewport: desktop
Description: Проверяет, что после выбора соцсети открывается список разделов сервиса.

Steps:
1. Открыть страницу BotHunter.
2. Выбрать соцсеть ВК.
3. Проверить, что открылась страница с группами разделов и карточками разделов.

Expected: Для выбранной соцсети отображается список разделов BotHunter.

## Test 4 - Типы тестирования внутри раздела открываются
Viewport: desktop
Description: Проверяет навигацию от раздела к типам тестирования.

Steps:
1. Открыть BotHunter -> ВК.
2. Нажать на раздел "Пользователи".
3. Проверить, что отображаются карточки типов тестирования.

Expected: Открывается страница раздела с доступными типами тестирования.

## Test 5 - Артефакты тестирования открываются на мобильной ширине
Viewport: mobile
Description: Проверяет, что на мобильном размере экрана можно дойти до страницы с артефактами тестирования.

Steps:
1. Открыть сайт на ширине 375x667.
2. Перейти BotHunter -> ТГ -> Пользователи.
3. Открыть тип "Функциональное тестирование".
4. Проверить наличие карточек артефактов.

Expected: На мобильной ширине страница артефактов отображается корректно и остается читаемой.
